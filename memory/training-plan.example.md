## Periodization overview
_Use base → build → peak → taper for a dated event goal. For an ongoing/non-event goal,
replace this with a rolling focus-block structure instead (e.g. alternating volume and
strength/durability blocks) — periodization toward a fixed date doesn't apply._
- **Base** — aerobic volume, climbing durability, consistency.
- **Build** — add threshold/VO2 and long climbing-specific rides.
- **Peak** — race-specific intensity, sharpen.
- **Taper** — reduce volume, keep intensity, arrive fresh.

## Standardized calendar event template
Use this shape for every training event written to Google Calendar, so descriptions stay
consistent across sessions. Google Calendar does NOT render Markdown in event
descriptions — `**bold**` and `- bullet` show up as literal characters. Use real HTML
tags instead (Calendar supports `<b>`, line breaks via `\n`, and a `•` character for list
items — avoid `<ul>`/`<li>`, they don't render reliably).

```
🚴 [Session type/name]

<b>Moving time:</b> range/upper/lower bound (e.g. "60–70 min", "≤90 min", "≥45 min")
<b>Total time (incl. breaks):</b> this is what sets the calendar event's actual start/end block
<b>Distance:</b> range/upper/lower bound (e.g. "28–32 km")
<b>Elevation:</b> range/upper/lower bound (e.g. "200–350 m")

<b>Structure (each phase with its own HR/effort target):</b>
• Warm-up: [duration] @ [HR/effort target]
• Main set: [reps/duration] @ [HR/effort target]
• Cool-down: [duration] @ [HR/effort target]

<b>Route notes:</b> terrain needed, known good segments, things to avoid (timing/traffic).
<b>Fueling/hydration (long rides only, >90 min):</b> carbs/hr target, hydration cadence.
<b>Misc notes:</b> optional — anything else relevant, filled in as needed.
```

Deliberately does NOT include a post-ride stretch/roll section — that's data-driven
(depends on what the ride actually stressed) and belongs in `/ride-analysis` output only,
not written into the calendar ahead of time. Pre-ride dynamic warm-up prep can still live
in the event description since it's decidable in advance.

Use the athlete's real weekday start times (see `athlete-profile.md`'s Schedule &
logistics section) rather than an arbitrary default when scheduling after-work sessions.
